# Web

### GET방식과 POST방식
> - **GET**   
>   -서버로부터 정보를 조회하기 위해 설계   
>   -URL뒤에 key=value형태로 전달하는 방식    
>   -길이 제한이 있으므로 전송 데이터에 한계가 있음   
>   -불필요한 요청을 제한하기 위해 요청이 캐시될 수 있음   
> - **POST**   
>   -리소스를 생성/변경하기 위해 설계   
>   -GET과 달리 전송해야될 데이터를 HTTP 메세지의 Body에 담아서 전송
